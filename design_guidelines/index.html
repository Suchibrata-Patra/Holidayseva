Foundations · colors.css

Color

All colors live as CSS variables on :root inside colors.css. Use the token — never a raw hex.

Brand

Primary--color-primary
Primary Light--color-primary-light
Primary Dark--color-primary-dark
Secondary--color-secondary
Accent--color-accent

Semantic

Success--color-success
Warning--color-warning
Error--color-error
Text Secondary--color-text-secondary
Muted--color-muted
ℹ

Always use var(--color-primary), never #FF385C. This keeps theming possible across the entire system.

css · colors.css 
:root {
  --color-primary:       #FF385C;
  --color-primary-light: #FF5A76;
  --color-primary-dark:  #D93B55;
  --color-secondary:     #222222;
  --color-accent:        #00A699;
  --color-success:       #008A05;
  --color-warning:       #C45508;
  --color-error:         #C13515;
}

Component · button.css

Button

Use Primary for the main CTA. Ghost for cancel or low-emphasis actions. Never two Primary buttons side-by-side.

Variants

live
html 
<button class="btn btn-primary">Reserve</button>
<button class="btn btn-secondary">Save</button>
<button class="btn btn-outline">Share</button>
<button class="btn btn-ghost">Cancel</button>
<button class="btn btn-primary" disabled>Disabled</button>

Sizes

live
html 
<button class="btn btn-primary btn-sm">Small</button>
<button class="btn btn-primary">Default</button>
<button class="btn btn-primary btn-lg">Large</button>
<button class="btn btn-primary btn-icon">★</button>
<button class="btn btn-primary btn-full">Full Width</button>
✓ Do
One Primary per view. Pair with Ghost or Outline for secondary options.
✕ Don't
Multiple Primary buttons side-by-side create noise and ambiguity.
