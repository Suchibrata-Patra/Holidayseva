01 — Setup

Getting Started

Two files. One CSS import. One script tag. No bundler, no npm, no config. Just HTML.

Single entry point: Always import ui-framework.css — it @imports all individual component CSS files automatically. Never import component CSS files individually.
Copy all files into your project
Place all CSS files, ui-framework.css, and airframe.js in the same directory (e.g., /assets/airframe/). Keep all files together — ui-framework.css uses relative @import paths.
Add the CSS to your <head>
One link tag for the master file. That's it — all components are included.
Add the JS before </body>
The script auto-initialises all interactive components (dropdowns, modals, calendars, wishlists) when the DOM is ready.
Use component class names in your HTML
No extra config needed. Just use the classes as documented below — the framework handles the rest.
Minimal HTML boilerplate HTML
html 
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />

  <!-- ✅ Import the MASTER file — loads all components -->
  <link rel="stylesheet" href="assets/airframe/ui-framework.css" />

  <!-- ❌ DON'T import individual CSS files directly -->
  <!-- <link rel="stylesheet" href="button.css" /> -->
</head>
<body>

  <!-- Your HTML using Airframe class names -->
  <button class="btn btn-primary">Reserve Now</button>

  <!-- ✅ Script before closing body tag -->
  <script src="assets/airframe/airframe.js"></script>
</body>
</html>

JavaScript

airframe.js

A single vanilla JS file. Drop it in with a <script> tag and all interactive components initialise automatically. No framework, no bundler, no config.

Auto-init on load: The script listens for DOMContentLoaded and inits dropdowns, modals, calendars, scroll-spy, calendar tabs, and wishlist buttons. You don't need to call anything yourself for standard use.
Script tag placement airframe.js
html
<body>
  <!-- ... your page content ... -->

  <!-- ✅ Place BEFORE closing </body> tag -->
  <script src="assets/airframe/airframe.js"></script>
</body>

Public API

All public methods are exposed on window.AirframeUI and also as global functions for use in inline onclick attributes.

Method Signature Description
openModal openModal(id) Show modal overlay by element ID. Locks body scroll, focuses close button.
closeModal closeModal(id) Hide modal by element ID. Restores body scroll.
toggleDropdown toggleDropdown(idOrEl) Toggle a dropdown open/closed. Pass the .dropdown element's ID or the element itself.
closeAllDropdowns closeAllDropdowns() Close every open dropdown on the page.
changeCount changeCount(type, delta, opts?) Increment/decrement a counter. type is 'adults'/'children'/'infants'. opts accepts {min, max}.
initCalendar initCalendar(el) Manually init a calendar wrapper element (auto-called on page load).
clearCalendar clearCalendar(el?) Clear selected date range. Without argument, targets #calendarEl.
toggleReview toggleReview(id, btn) Toggle truncated/expanded state of a review body. Pass the body element ID and the button element.
doCopy doCopy(btn) Copy code from nearest .doc-code pre to clipboard. Shows "Copied!" feedback.
formatDate formatDate(date) Returns a locale-formatted date string (e.g. "12 Mar 2025") from a JS Date object.

Events & Data Attributes

Airframe emits custom events and recognises special data attributes for integration with your own code.

Event / Attribute Where Description
calendar:change Event on .calendar-wrapper Fires when a date is selected. event.detail contains { start: Date, end: Date|null }. Bubbles.
data-booking Ancestor of calendar Marks the booking context. The calendar searches upward for this to find and update data-booking-checkin / data-booking-checkout text.
data-booking-checkin Span inside booking card Text content auto-updated by calendar when check-in date is selected.
data-booking-checkout Span inside booking card Text content auto-updated by calendar when check-out date is selected.
data-wishlist Any button Auto-inited toggle: adds .is-wishlisted class and changes color to primary red on click. Sets aria-pressed.
data-calendar Wrapper div Alternative to .calendar-wrapper class for triggering calendar auto-init.
Listening to calendar changes JS
javascript
const cal = document.getElementById('calendarEl');

cal.addEventListener('calendar:change', (e) => {
  const { start, end } = e.detail;

  if (start && end) {
    const nights = Math.round((end - start) / 86_400_000);
    console.log(`${nights} nights selected`);
  }
});
Programmatic modal from JS JS
javascript
// Open a modal
