HSDatepicker

A fully-featured date range picker with dual-calendar layout, hover range preview, flexible date chips, and auto-positioning. Works with any trigger element via a simple JS constructor.

✓ Vanilla JS Range & Single Auto-positioning No Dependencies

Installation

Drop two files into your project. No build tools or dependencies needed.

/Users/suchibratapatra/Documents/Holidayseva/Assets/js/hs-datepicker.js
HTML — include in <head> and before </body>

<!-- Before </body> -->

Usage

Three ways to use HSDatepicker — HTML attribute, JS constructor, or programmatic control.

1. HTML Trigger Structure

The trigger must be a non-button element (use <div>) if it contains a nested clear button.

Recommended trigger HTML
<!-- ✅ Use a <div> when nesting the clear button -->
<div id="myTrigger" class="hs-dp-trigger">
  <div>
    <span class="hs-dp-trigger-label">When</span>
    <span class="hs-dp-trigger-value">Add dates</span>
  </div>
  <button class="hs-dp-trigger-clear" style="display:none">
    <!-- × icon -->
    <svg viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2.5">
      <circle cx="12" cy="12" r="10"/>
      <path d="M15 9l-6 6M9 9l6 6"/>
    </svg>
  </button>
</div>

2. Auto-init via data-hs-datepicker

Automatic — no JS needed
<div id="myTrigger"
     class="hs-dp-trigger"
     data-hs-datepicker
     data-hs-datepicker-options='{"range":true,"position":"bottom-left"}'>
  <!-- trigger inner HTML -->
</div>

3. JS Constructor (recommended for callbacks)

Full constructor with all options
const picker = new HSDatepicker('#myTrigger', {
  range:      true,
  position:  'bottom-left',   // 'bottom-left' | 'bottom-right' | 'bottom-center'
  minDate:   new Date(),        // default: today
  maxDate:   null,              // no upper limit by default
  flexChips: [
    'Exact dates', '± 1 day', '± 2 days',
    '± 3 days',   '± 7 days', '± 14 days'
  ],
  onChange: ({ startDate, endDate }) => {
    console.log('Start:', startDate);
    console.log('End:',   endDate);
  }
});

Options

Passed as the second argument to the constructor, or as JSON in data-hs-datepicker-options.

Option Type Default Description
range boolean true Enable range selection (start + end). Set false for single-date mode — picker closes on first click.
position string 'bottom-left' Panel alignment relative to trigger. Values: 'bottom-left', 'bottom-right', 'bottom-center'.
minDate Date new Date() Earliest date the user can select. Past dates are greyed out and struck through by default.
maxDate Date null Latest selectable date. null means no upper limit.
flexChips string[] ['Exact dates', …] Array of labels for the flexibility chips in the footer. The first chip is active by default. Pass an empty array to hide the footer chips.
onChange Function null Callback fired when dates change. Receives { startDate: Date|null, endDate: Date|null }. Also fires on clear().

Methods

All public methods available on the HSDatepicker instance.

picker.open() void
Opens the date picker panel and positions it below the trigger element. Also adds hs-active class to the trigger.
picker.close() void
Closes the panel and removes the hs-active class from the trigger.
picker.toggle() void
Toggles open/close state. Equivalent to calling open() or close() depending on current state.
picker.clear() void
Resets both start and end dates to null, re-renders the calendars, updates the trigger display, and fires the onChange callback.
picker.getDates() { startDate, endDate }
Returns the currently selected dates as an object: { startDate: Date|null, endDate: Date|null }.
picker.setDates(start, end?) void
Programmatically sets the date range. Navigates the calendar view to the start month and updates the trigger label. 
const start = new Date('2026-04-10');
const end   = new Date('2026-04-15');
picker.setDates(start, end);

Events

In addition to the onChange callback, HSDatepicker fires a native custom DOM event on the trigger element.

hs:datechange
Fired on the trigger element whenever dates change (selection or clear). Use event.detail to access startDate and endDate. 
document.getElementById('myTrigger')
  .addEventListener('hs:datechange', (e) => {
    const { startDate, endDate } = e.detail;
    console.log(startDate, endDate);
  });

.hs-dp-day--end day cell Range end date — dark filled.

Theming

Override CSS custom properties to match your brand. All colours, fonts, and shadows flow from the root tokens.

Override design tokens in your own CSS
/* In your stylesheet, after importing hs-datepicker.css */
:root {
  /* Brand colour — affects selected days & active chips */
  --hs-color-text:   #1A1A2E;   /* dark navy instead of #222 */
  --hs-color-muted:  #8E8E93;

  /* Range highlight background */
  /* Override directly in CSS: */
}

/* Custom range colour example */
.hs-dp-day--in-range {
  background: #FFF0F3 !important;
}

.hs-dp-day--start,
.hs-dp-day--end {
  background: #FF385C !important;
}
🎨
Tip: The component inherits --hs-color-text and --hs-color-muted from holidayseva_framework.css. If you use the framework, you only need to override those root tokens — all components update automatically.

Notes & Known Constraints

⚠️
Never nest a <button> inside a <button>. If your trigger is a <button> and you need a clear ×, the inner button will be silently dropped by the browser. Use <div class="hs-dp-trigger"> as shown in the usage examples above.
⚠️
Panel is appended to <body>. The overlay and panel are injected into document.body at construction time, not inside the trigger's parent. This avoids overflow/clip issues but means the panel is not scoped to a specific container.
📌
Window resize: The panel position is calculated on open(), not live-tracked. If the page layout shifts after opening, call picker.close() and reopen to reposition.
