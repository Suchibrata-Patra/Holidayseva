Build with clarity and consistency

A complete, token-driven design system for Airbnb-style interfaces. Every component is built from the same foundations — color, type, spacing — so your UI stays coherent at any scale.

Foundations · Color

Color

All colors are exposed as CSS custom properties on :root. Use semantic tokens — never hardcode hex values in components.

Brand Palette

Primary --color-primary #FF385C
Primary Light --color-primary-light #FF5A76
Primary Dark --color-primary-dark #D93B55
Secondary --color-secondary #222222
Accent --color-accent #00A699

Semantic Colors

Success --color-success
Warning --color-warning
Error --color-error
Text Secondary --color-text-secondary
Muted --color-muted
ℹ

Always use semantic tokens like --color-text-primary in components, not raw hex values. This ensures global theming and dark mode support work without component-level changes.

css 
/* colors.css — always import this first */
:root {
  --color-primary:       #FF385C;
  --color-primary-light: #FF5A76;
  --color-primary-dark:  #D93B55;
  --color-secondary:     #222222;
  --color-accent:        #00A699;
  --color-success:       #008A05;
  --color-error:         #C13515;
}

/* Usage in a component */
.my-button {
  background: var(--color-primary);
  color:      var(--color-text-inverse);
}

Foundations · Border Radius

Border Radius

Consistent corner radii give the design a cohesive, friendly character. Larger surfaces use larger radii; inline elements use tighter ones.

sm · 4px
md · 8px
lg · 12px
xl · 16px
2xl · 24px
pill · ∞
circle · 50%
css 
:root {
  --radius-sm:   4px;
  --radius-md:   8px;
  --radius-lg:   12px;
  --radius-xl:   16px;
  --radius-2xl:  24px;
  --radius-pill: 9999px;
}

/* Usage */
.card   { border-radius: var(--radius-xl)   }  /* Large surface */
.badge  { border-radius: var(--radius-pill) }  /* Inline label */
.button { border-radius: var(--radius-md)   }  /* Interactive  */
Components · Button

Button

Buttons communicate actions. Use Primary for the most important action on screen, Secondary for supporting actions, and Ghost for tertiary or destructive actions in low-emphasis contexts.

Variants

Preview
html 
<button class="btn btn-primary">Reserve</button>
<button class="btn btn-secondary">Save</button>
<button class="btn btn-outline">Share</button>
<button class="btn btn-ghost">Cancel</button>
<button class="btn btn-primary" disabled>Disabled</button>

Sizes

Preview
html 
<!-- Sizes: add btn-sm or btn-lg -->
<button class="btn btn-primary btn-sm">Small</button>
<button class="btn btn-primary">Default</button>
<button class="btn btn-primary btn-lg">Large</button>
<button class="btn btn-primary btn-icon">★</button>
<button class="btn btn-primary btn-full">Full Width</button>
✓ Do
Use one Primary button per view. It signals the clearest path forward. Pair with Ghost or Outline for secondary options.
✕ Don't
Don't use multiple Primary buttons side-by-side. It creates visual noise and makes users uncertain about which action to take.
