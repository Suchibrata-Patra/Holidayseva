Foundations · colors.css

Color

All colors live as CSS custom properties on :root. Always use the token — never a raw value. This keeps the system themeable end-to-end.

Brand
Primary--color-primary
Primary Light--color-primary-light
Primary Dark--color-primary-dark
Secondary--color-secondary
Accent--color-accent
Semantic
Success--color-success
Warning--color-warning
Error--color-error
Secondary Text--color-text-secondary
Muted--color-muted
ℹ

Always reference var(--color-primary), never the raw hex #FF385C. This ensures every component participates in theming seamlessly.

css · colors.css 
:root {
  --color-primary:       #FF385C;
  --color-primary-light: #FF5A76;
  --color-primary-dark:  #D93B55;
  --color-secondary:     #222222;
  --color-accent:        #00A699;
  --color-success:       #008A05;
  --color-warning:       #C45508;
  --color-error:         #C13515;
}

Component · button.css

Button

Use Primary for the main call to action. Ghost for cancel or low-emphasis actions. Never place two Primary buttons side-by-side.

Variants
live preview
html 
<button class="btn btn-primary">Reserve</button>
<button class="btn btn-secondary">Save</button>
<button class="btn btn-outline">Share</button>
<button class="btn btn-ghost">Cancel</button>
<button class="btn btn-primary" disabled>Disabled</button>
Sizes
live preview
✓ Do
One Primary per view. Pair with Ghost or Outline for secondary actions.
✕ Don't
Multiple Primary buttons side-by-side create noise and ambiguity.
